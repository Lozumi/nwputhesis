\section{UAV-DualCog-Train训练集构建}

\subsection{构建目标与场景隔离}

第3章的评测结果表明，当前多模态大模型在 UAV-DualCog 上的主要问题是难以同时保持答案正确、证据可靠、跨媒介稳定以及自我状态认知与环境情况认知之间的协调发展。尤其是在图像任务中，模型常表现为选项判断相对较强、空间证据明显偏弱；在双认知维度上，自我状态认知长期落后于环境情况认知。因此，本文在测试基准之外进一步构建了面向后训练优化的图像任务训练集 UAV-DualCog-Train，并据此设计“监督微调（Supervised Fine-Tuning, SFT）+联合群组相对策略优化（Group Relative Policy Optimization, GRPO）”的能力优化方法。

UAV-DualCog-Train 使用与测试集严格隔离的训练场景构建，包含 \texttt{env\_1}、\texttt{env\_3}、\texttt{env\_5}、\texttt{env\_15}、\texttt{env\_18} 和 \texttt{env\_22} 六个场景，具体训练/测试场景划分见\autoref{tab:app-scene-split}。训练样本覆盖地标参照位置推理、未来观察预测、自我参照位置推理和地标驱动动作决策四类任务，对应自我状态认知与环境情况认知两条能力主线。当前优化阶段优先从图像任务起步，主要基于两点考虑：其一，第3章已经表明图像侧最集中地暴露了“语义判断与空间证据脱节”的问题；其二，图像任务适合将“离散答案+目标框证据”统一写成结构化监督目标，为后续监督微调和奖励优化共用同一套输出解析接口提供便利。

\subsection{训练记录格式与样本组织}

为保证训练目标与评测协议一致，训练记录与真实多模态输入直接对齐。每条样本会被转换为统一的 \texttt{prompt}、\texttt{completion} 和 \texttt{images} 三元结构，其中 \texttt{prompt} 保留 benchmark 原有的系统提示与用户提示，\texttt{images} 以与评测阶段一致的顺序存放图像路径，\texttt{completion} 则使用结构化 JSON 监督目标：
\begin{verbatim}
{
  "answer_option_id": "B",
  "bbox_xyxy_norm": [0.112193, 0.400801, 0.382753, 0.698569]
}
\end{verbatim}
对于地标参照位置推理、自我参照位置推理和地标驱动动作决策三类任务，训练输入使用“地标参考图 + 查询图像”两图组合；对于未来观察预测任务，训练输入则保留“地标参考图 + 候选图像序列”的多图结构。与此同时，样本记录中还会缓存 \texttt{answer\_bbox\_xyxy\_1000} 等辅助字段，供奖励计算阶段直接复用。这样的设计使训练阶段围绕“结构化答案输出、空间证据可核验”的目标展开。训练数据字段、划分策略和数据准备伪代码见\autoref{app:train-dataset-construction}，训练入口与关键实现片段见\autoref{app:optimization-entry}。

\subsection{训练集规模}

UAV-DualCog-Train-V1 的规模为
\[
234 \times 4 \times 2 \times 6 = 11232,
\]
即 234 个有效地标、4 类任务、2 档难度、每档难度 6 个问答样本。训练集与验证集采用稳定轮转划分策略，在每个 \texttt{(task\_type, difficulty)} 桶内按样本编号排序后，每 10 条样本取 1 条进入验证集，其余进入训练集。相比完全随机划分，这种做法能够在不同任务和难度子集之间保持更稳定的 9:1 比例，同时避免训练集与验证集在样本编号层面发生重叠。

\section{能力优化方法}

\subsection{多模态监督微调（SFT）}

能力优化的第一阶段采用多模态监督微调（Supervised Fine-Tuning, SFT）。本阶段的目标就是给模型创建稳定的任务内输出接口，使它能够围绕 UAV-DualCog 的图像任务产生结构化的答案和目标框结果。在训练过程中，图像会被 \texttt{AutoProcessor} 编码成视觉张量参与到前向和反向传播中。因此SFT阶段的主要作用就是使得模型在多个图像输入的情况下能够稳定的给出可以解析的结构化的答案。

本文选择 \texttt{Qwen/Qwen3.5-4B} 作为当前能力优化的基座模型，主要基于三方面考虑。第一，第3章结果显示 Qwen 3.5-4B 已具备一定图像空间问答基础，其图像总体准确率为 39.0\%。第二，该模型在视频总体准确率和自我状态认知上仍有明显短板，具备清晰的优化空间。第三，4B 规模适合在现有四张 RTX 4090 的算力条件下进行多轮参数高效实验，并便于后续补充消融研究。

在方法上，SFT 阶段使用 \texttt{AutoModelForImageTextToText}、\texttt{AutoProcessor} 和 \texttt{trl.SFTTrainer} 完成多模态低秩适配（Low-Rank Adaptation, LoRA）微调。LoRA只对注意力和前馈网络的关键投影层进行可训练的低秩适配，从而在有限的显存和训练步数下完成任务适配。训练时使用  \texttt{completion\_only\_loss}，即只对助手端的 JSON 输出部分计算损失，将格式学习、答案学习和证据学习统一绑定到同一个训练接口上。为了同时保证图像细节以及显存的稳定性，在加载训练图像的时候引入了 \texttt{max\_image\_side} 的控制项，将长边设为720，保留较多空间细节。具体训练入口、配置与关键代码见\autoref{app:optimization-entry}。


\subsection{联合群组相对策略优化（GRPO）}

在完成结构化监督学习之后，本文进一步引入联合群组相对策略优化（Group Relative Policy Optimization, GRPO）进行第二阶段优化。该阶段直接在 SFT 检查点基础上，以统一奖励函数同时约束输出格式、离散答案和目标框质量。多模态训练链路已经能够稳定支持真实图像输入和 JSON 输出解析，因此奖励优化可以围绕最终任务目标进行联合校正。

联合 GRPO 的总体奖励形式为
\begin{equation}
R = w_f R_{\text{format}} + w_s R_{\text{mcq}} + w_b R_{\text{bbox}},
\end{equation}
其中 \(R_{\text{format}}\) 衡量模型输出能否被解析为规范 JSON，\(R_{\text{mcq}}\) 衡量离散答案是否正确，\(R_{\text{bbox}}\) 衡量预测目标框与真实目标框的一致性。奖励实现优先解析 \texttt{\{"answer\_option\_id", "bbox\_xyxy\_norm"\}} 形式的 JSON 输出；目标框奖励则基于归一化坐标上的 GIoU 计算，并映射到 \([0,1]\) 区间。该设计与第3章的分析结论对应，即同时缩小“答得对”与“证据也对”之间的差距。

\subsection{训练环境与正式流程}

能力优化实验在一台本地服务器上进行，硬件环境包括 \(2\times\) AMD EPYC 7452 CPU、503 GiB 内存和 \(4\times\) NVIDIA GeForce RTX 4090 GPU。训练过程采用 PyTorch、Transformers、PEFT 和 TRL 等框架完成单机多卡多模态 LoRA-SFT 与联合 GRPO 优化，详细环境配置见\autoref{tab:app-train-hardware}。

本文将训练流程组织为“数据准备、多模态 SFT、联合 GRPO、任务级评测”四个环节。首先，将 UAV-DualCog-Train 图像任务清单转换为 train/valid JSONL，并保留提示词、真实图像路径、标准 JSON completion 和目标框辅助字段；其次，SFT 阶段从 Qwen 3.5 4B 初始化，通过真实图像输入和 completion-only loss 学习稳定的结构化输出；再次，联合 GRPO 阶段以 SFT 模型为初始化模型，对同一输入生成多候选答案，并依据格式、答案和目标框奖励进行组相对优化；最后，将优化模型接入 UAV-DualCog-V1 测试集，评估图像任务直接收益和视频任务迁移效果。训练运行记录和完整性核查结果见\autoref{tab:app-train-run-summary}，训练入口、流程伪代码和奖励实现见\autoref{app:optimization-entry}。

环境情况认知的迁移结果呈现另一种形态。环境可见性推理的 Count Accuracy 从 33.89\% 下降到 31.35\%，Segment F1@0.5 从 45.18\% 下降到 24.23\%，Mean tIoU 从 51.99\% 下降到 29.74\%。结合图像任务结果看，这一变化并不说明优化模型没有学到地标识别能力。相反，优化模型在自我参照位置推理和地标驱动动作决策中的选项准确率、BBox Acc@0.5 和 mIoU 均显著提升，说明其对静态图像以及逐帧观测中的地标外观、位置和目标框证据更加敏感。问题在于，环境可见性推理并不只要求模型在某一帧中“找到地标”，而是要求模型在整段视频中持续追踪同一地标，并判断它在视角变化、遮挡、距离变化和画面边缘截断条件下是否仍然构成连续可见。

因此，图像侧证据能力提高会加强模型对局部可见帧的检测，但是不会同时加强跨帧连续性建模。环境可见性任务要求模型结合自身的飞行状态来判断画面的变化原因，比如无人机是在接近、远离、环绕还是上升观察；还要将相邻可见帧整合成完整的时段，并且不能把短暂遮挡或者视角抖动当作新的出现次数。目前的训练奖励只对静态 BBox 和选项答案进行约束，没有直接约束跨帧目标一致性、可见区间合并和出现次数统计，因此优化模型更容易发现局部帧中的目标，但是很难将局部证据稳定组织成视频级可见事件。该现象也表明，无人机双认知能力不能只停留在单帧环境识别上，而应该把环境情况认知和自我状态认知结合起来，用飞行状态解释连续观测中目标可见性的变化。

从两类视频任务对比来看，图像训练带来的迁移收益主要发生在“地标相关动作识别”而非“地标持续可见性整合”上。前者可以依靠地标相对位置以及画面运动方向来找到与自身运动方式一致的片段，空间证据加强可以马上辨别出动作的边框；后者需要更高的时序记忆、跨帧目标追踪和片段融合能力，不能只用静态的目标框训练得到。后续若要改善环境可见性推理，需要进一步加入视频任务训练样本、跨帧目标一致性约束或者时间区间级奖励，使得模型不仅可以逐帧识别地标，还可以将无人机自身的运动状态与连续可见的过程结合起来。

\section{本章小结}

本章围绕第3章暴露出的能力短板，系统介绍了 UAV-DualCog 的能力优化方法与实验结果。首先，本文构建了与测试场景严格隔离的 UAV-DualCog-Train 图像任务训练集，并将其转换为包含提示词、真实图像路径和标准 JSON completion 的多模态训练记录。该训练集在任务类型、难度等级和双认知分组上保持平衡，为模型学习“答案+目标框”的结构化输出提供了数据基础。

其次，本文完成了基于 Qwen 3.5 4B 的两阶段能力优化流程：多模态 LoRA-SFT 用于建立稳定的结构化答案输出能力，联合 GRPO 则通过格式奖励、答案奖励和目标框奖励进一步强化语义-证据一致性。训练过程完整产出可评测模型。任务级评测显示，优化模型在图像任务上获得显著直接收益，图像选项准确率、BBox Acc@0.5 和 mIoU 分别提升 20.78、51.76 和 32.66 个百分点；在视频任务上，全局准确率提升 9.91 个百分点，但收益主要集中在自我行为识别和时间定位，环境可见性推理仍出现下降。由此可见，证据感知图像训练能够有效补强图像双认知能力并产生部分跨媒介迁移，但可靠无人机多模态系统仍需要进一步提升视频时序整合和跨媒介证据协同能力。
