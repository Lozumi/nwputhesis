\subsection{四阶段构建流程伪代码}

\begin{verbatim}
Input: AerialVLN scenes and task configuration
Output: UAV-DualCog benchmark with image/video evidence labels

for each selected scene do
    Stage 1: scan scene and collect RGB / semantics / LiDAR / poses
    fuse observations into a scene-level semantic point cloud

    Stage 2: cluster landmark candidates from semantic point cloud
    render multi-view landmark candidates
    conduct human review and semantic labeling
    export validated landmark assets

    Stage 3: bind landmarks with flight behavior templates
    generate trajectories and first-person videos
    record temporal evidence for actions and landmark visibility
    export self-aware and environment-aware video tasks

    Stage 4: sample landmark-centered reference view
    sample UAV observation view or future-view candidates
    generate image QA tasks and normalized bbox evidence
end for
\end{verbatim}

阶段3的关键在于为同一条轨迹同时建立高层行为语义与底层时间证据。阶段4的关键在于同步生成选项答案和目标框证据，而不是先生成问题再事后补充定位框。因此，UAV-DualCog 的任务构建天然具备“语义答案 + 显式证据”的联合评测属性。

\subsection{图像任务完整中文提示词模板}

下列模板为实际英文评测提示词的中文等价版本。模板中的占位字段由样本构建程序根据具体任务自动替换，主要包括：
\begin{itemize}
    \item \texttt{reference\_object\_view}：参考立面或参考视角字段；
    \item \texttt{landmark\_description}：目标地标文本描述字段；
    \item \texttt{behavior\_instance}：飞行动作或轨迹变化字段；
    \item \texttt{options\_text}：候选选项字段。
\end{itemize}

\paragraph{地标参照位置推理。}
\textbf{系统提示词：} 你是一个 UAV agent。你将获得一张包含目标地标的参考视图和一张当前第一视角图像。图像1为参考视图，显示目标地标及其目标框；图像2为当前视图。图像1中可见的立面定义了地标中心坐标系下的参考方向。你必须根据该参考方向判断自己在图像2中相对于地标的位置，并同时定位图像2中的目标地标。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；只能使用用户提示词中给出的选项编号；目标框必须使用归一化 \([x_1,y_1,x_2,y_2]\) 坐标。
\begin{verbatim}
{"answer_option_id":"A","bbox_xyxy_norm":[0.1,0.2,0.3,0.4]}
\end{verbatim}
\textbf{用户提示词：} 图像1显示地标中心坐标系中 \texttt{\{reference\_object\_view\}} 立面的 \texttt{\{landmark\_description\}}。基于这一参考，图像2中你相对于该地标处于什么位置？请选择一个选项，并返回图像2中该地标的归一化目标框。选项为：\texttt{\{options\_text\}}。解析字段为 \texttt{answer\_option\_id} 和 \texttt{bbox\_xyxy\_norm}。

\paragraph{未来观察预测。}
\textbf{系统提示词：} 你是一个 UAV agent。你将获得一张包含目标地标的参考视图和四张候选未来视图。图像1为参考视图，显示目标地标及其目标框；图像2到图像5为围绕同一地标的候选未来视图。图像1中可见的立面定义了地标中心坐标系下的参考方向。你必须根据给定轨道运动选择唯一正确的未来视图，并在所选候选图像中定位目标地标。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；目标框必须使用归一化坐标。
\begin{verbatim}
{"answer_option_id":"A","bbox_xyxy_norm":[0.1,0.2,0.3,0.4]}
\end{verbatim}
\textbf{用户提示词：} 图像1显示地标中心坐标系中 \texttt{\{reference\_object\_view\}} 立面的 \texttt{\{landmark\_description\}}。执行 \texttt{\{behavior\_instance\}} 后，哪一张候选图像是对应的未来视图？请选择一个选项，并返回所选图像中该地标的归一化目标框。选项为：A. 图像2；B. 图像3；C. 图像4；D. 图像5。解析字段为 \texttt{answer\_option\_id} 和 \texttt{bbox\_xyxy\_norm}。

\paragraph{自我参照位置推理。}
\textbf{系统提示词：} 你是一个 UAV agent。你将获得目标地标参考视图和当前第一视角图像。图像1为参考视图，图像2为当前视图。你需要判断目标地标在图像2中相对于自己前进方向的位置，并同时定位该地标。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；只能使用用户提示词中给出的选项编号；目标框必须使用归一化坐标。
\begin{verbatim}
{"answer_option_id":"A","bbox_xyxy_norm":[0.1,0.2,0.3,0.4]}
\end{verbatim}
\textbf{用户提示词：} 图像1显示地标中心坐标系中 \texttt{\{reference\_object\_view\}} 立面的 \texttt{\{landmark\_description\}}。在图像2中，该地标相对于你的前进方向位于哪里？请选择一个选项，并返回图像2中该地标的归一化目标框。选项为：\texttt{\{options\_text\}}。解析字段为 \texttt{answer\_option\_id} 和 \texttt{bbox\_xyxy\_norm}。

\paragraph{地标驱动动作决策。}
\textbf{系统提示词：} 你是一个 UAV agent。你将获得目标地标参考视图和当前第一视角图像。你需要判断为了接近该地标应向哪个方向飞行，并同时定位当前图像中的目标地标。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；只能使用用户提示词中给出的选项编号；目标框必须使用归一化坐标。
\begin{verbatim}
{"answer_option_id":"A","bbox_xyxy_norm":[0.1,0.2,0.3,0.4]}
\end{verbatim}
\textbf{用户提示词：} 图像1显示地标中心坐标系中 \texttt{\{reference\_object\_view\}} 立面的 \texttt{\{landmark\_description\}}。在图像2中，为了接近该地标，你应该向哪个方向飞行？请选择一个选项，并返回图像2中该地标的归一化目标框。选项为：\texttt{\{options\_text\}}。解析字段为 \texttt{answer\_option\_id} 和 \texttt{bbox\_xyxy\_norm}。

\subsection{视频任务完整中文提示词模板}

\paragraph{复合飞行行为识别。}
\textbf{系统提示词：} 你是一个 UAV agent。请观看自己的第一视角飞行视频，识别视频中曾经执行过的高层飞行动作。用户提示词会提供候选选项。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；只能使用用户提示词中出现的 \texttt{option\_id}；每个答案项必须包含选项编号及其对应秒级时间区间；若多个候选动作确实出现，需要全部返回。
\begin{verbatim}
{"answers":[{"option_id":"A","intervals_sec":[[0.0,1.2]]}]}
\end{verbatim}
\textbf{用户提示词：} 在该第一视角飞行视频中，你曾经执行过以下哪些动作？请选择一个或多个选项，并返回对应的秒级时间区间。选项为：\texttt{\{options\_text\}}。解析字段为 \texttt{answers[].option\_id} 和 \texttt{answers[].intervals\_sec}。

\paragraph{原子飞行行为识别。}
\textbf{系统提示词：} 你是一个 UAV agent。请观看自己的第一视角飞行视频，识别视频中执行过的低层原子飞行动作。用户提示词会提供候选选项。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；只能使用用户提示词中出现的 \texttt{option\_id}；每个答案项必须包含选项编号及其对应秒级时间区间；若多个候选动作确实出现，需要全部返回。
\begin{verbatim}
{"answers":[{"option_id":"A","intervals_sec":[[2.1,4.6]]}]}
\end{verbatim}
\textbf{用户提示词：} 在该第一视角飞行视频中，你执行了以下哪一个动作？请选择一个选项，并返回对应的秒级时间区间；若多个候选动作确实出现，则需要全部返回。选项为：\texttt{\{options\_text\}}。解析字段为 \texttt{answers[].option\_id} 和 \texttt{answers[].intervals\_sec}。

\paragraph{地标可见次数与时间定位。}
\textbf{系统提示词：} 你是一个 UAV agent。请观看自己的第一视角飞行视频，并结合一张目标地标参考图判断该地标何时可见。你将获得一个第一视角飞行视频、一张显示目标地标及目标框的参考图像，以及该地标的简短文本描述。只允许返回一个合法 JSON 对象，不得输出 Markdown、解释或额外文本；需要统计目标地标分几次可见，并返回完整可见时间区间。
\begin{verbatim}
{"visible_count":2,"visible_intervals_sec":[[0.0,9.1],[17.7,26.8]]}
\end{verbatim}
\textbf{用户提示词：} 参考图显示 \texttt{\{landmark\_description\}}。该地标在你的第一视角飞行视频中出现了几次？请返回完整可见时间区间，单位为秒。解析字段为 \texttt{visible\_count} 和 \texttt{visible\_intervals\_sec}。

\subsection{解析与有效性规则}

所有任务均要求严格 JSON 输出。包含 Markdown 代码块、额外自然语言解释、字段缺失或字段类型错误的结果会被视为格式无效。图像任务只有在选项与目标框均可解析时才进入完整评分；视频任务则将语义识别或可见次数与时间区间质量分开计算，从而避免把“识别正确但证据错误”的输出误判为完整成功。

\subsection{奖励函数关键代码片段}
\label{app:reward-code}

下列代码片段节选自奖励实现，展示本文如何优先解析 JSON 结构化输出，并将格式合法性、答案正确性和目标框一致性组合为统一奖励。
\begin{verbatim}
def reward_format(text: str) -> float:
    return 1.0 if (
        parse_answer(text) is not None
        and parse_bbox_norm(text) is not None
    ) else 0.0

def reward_mcq(pred_text: str, gt_answer: str) -> float:
    pred = parse_answer(pred_text)
    if pred is None:
        return 0.0
    return 1.0 if pred == str(gt_answer).strip().upper() else 0.0

def reward_bbox(pred_text: str, gt_bbox_norm):
    pred_bbox = parse_bbox_norm(pred_text)
    if pred_bbox is None:
        return 0.0
    giou = giou_norm(pred_bbox, gt_bbox_norm)
    return max(0.0, min(1.0, (giou + 1.0) * 0.5))

def total_reward(pred_text, gt_answer, gt_bbox_norm, weights):
    return (
        weights.w_format * reward_format(pred_text)
        + weights.w_mcq * reward_mcq(pred_text, gt_answer)
        + weights.w_bbox * reward_bbox(pred_text, gt_bbox_norm)
    )
\end{verbatim}

该实现与第4章中的奖励公式一一对应，保证能力优化目标并非停留在方法设想层面，而是直接进入了最终训练与任务级评测流程。
